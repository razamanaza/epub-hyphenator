## Architecture

**What it does:** Accepts an EPUB file upload + language selection, runs the external `epub-hyphen` CLI on it, and returns the hyphenated EPUB for download.

**Stack:** React 19 + TanStack Start (SSR/API framework built on Nitro/Vite) + TypeScript + Tailwind CSS 4.

**Request flow:**
1. `src/routes/index.tsx` — home page renders `<UploadForm>`
2. `src/components/UploadForm.tsx` — handles file selection, client-side validation (type, 50MB limit), and form submission
3. `POST /api/process-epub` handled by `src/routes/api/process-epub.ts` — saves upload to a temp file, shells out to `epub-hyphen -l <lang> input.epub -o output.epub`, reads the result, cleans up, and returns the binary
4. Frontend receives the binary blob and triggers a browser download

**Route tree** (`src/routeTree.gen.ts`) is auto-generated by TanStack Router — do not edit manually.

**Validation:** Zod is used in the API route to validate multipart form data. Client-side validation mirrors server-side rules.

**External dependency:** The `epub-hyphen` npm package (plus `hyphenation.ru` for Russian) must be installed globally — the Docker image handles this in its production stage.

## Code style

- No semicolons, single quotes, trailing commas (Prettier enforced)
- Strict TypeScript — `noUnusedLocals`, `noUnusedParameters`
- Function names are verbs; value/variable names are nouns
- ESLint uses TanStack config (`eslint.config.js`)
